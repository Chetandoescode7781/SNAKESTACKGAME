\section{External Libraries}
This project mainly uses standard web technologies such as HTML5, CSS, and JavaScript. These technologies are enough to build interactive browser-based games without depending on heavy external libraries.

On the backend side, Python is used along with the Flask framework. Flask is a lightweight web framework that helps in handling HTTP requests and building APIs easily. It is suitable for small projects like this where only basic data handling is required.

The decision to avoid additional libraries was made to keep the project simple and focused on understanding core concepts like game logic, rendering, and client-server communication.

\section{Design \& Architecture}

\subsection{System Overview}
The system follows a simple client-server architecture. It consists of a frontend that runs in the browser and a backend that runs locally using Flask.

The frontend is responsible for rendering the game and handling user input. The backend is responsible for storing the score data.

\subsection{Frontend Design}
The frontend is implemented using HTML5 Canvas and JavaScript. The canvas element acts as the main drawing area where the snake and food are displayed.

JavaScript handles:
\begin{itemize}
    \item Game loop execution
    \item Snake movement
    \item Collision detection
    \item Food generation
\end{itemize}

\subsection{Backend Design}
The backend is implemented using Flask. It exposes a simple API endpoint that receives the player's score.

When the game ends, a fetch request is sent from the frontend to the backend. The backend then stores the data in a text file.

\subsection{Data Flow}
The overall data flow in the system can be described as follows:

User input $\rightarrow$ JavaScript logic $\rightarrow$ Canvas rendering $\rightarrow$ Game Over $\rightarrow$ Fetch API $\rightarrow$ Flask server $\rightarrow$ history.txt

This flow ensures proper separation of concerns between the frontend and backend.

\section{Algorithm and Logic}

\subsection{Game Loop}
The game runs using a loop that repeatedly updates and redraws the game state.

\textbf{Pseudo Code:}
\begin{verbatim}
while game is running:
    update snake position
    check collisions
    check food consumption
    update score
    redraw canvas
\end{verbatim}

\subsection{Collision Detection}
Collision detection is done by checking:
\begin{itemize}
    \item If the snake hits the wall
    \item If the snake hits itself
\end{itemize}

If a collision is detected, the game ends.

\subsection{Food Generation}
Food is generated at random positions within the grid. Care is taken to ensure that food does not spawn on the snake's body.

\section{Installation \& Setup}

To run this project, the following steps are required:

\begin{itemize}
    \item Install Python 3.x
    \item Install Flask using pip
    \begin{verbatim}
pip install flask
    \end{verbatim}
    \item Navigate to the project folder
    \item Run the server using:
    \begin{verbatim}
python app.py
    \end{verbatim}
\end{itemize}

The project works on modern browsers such as Chrome and Edge.

\section{Usage}

After starting the server, open a browser and go to:

\begin{verbatim}
http://127.0.0.1:5000
\end{verbatim}

Enter your name and select difficulty. Use arrow keys or WASD to control the snake. Press spacebar to pause or resume the game.

The objective is to collect food and achieve the highest possible score.

\section{Retrospective}

\subsection{Challenges Faced}
Several challenges were faced during development. One challenge was implementing smooth movement of the snake. Another challenge was correctly detecting collisions in all scenarios.

Backend integration was also slightly difficult, especially sending and receiving data properly.

\subsection{Solutions}
These problems were solved by testing different approaches and debugging step by step. Console logs were used to track values and identify issues.
